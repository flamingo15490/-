本人的定位：有一定的语法基础，但是没有修过算法。我是大二的学生，大一的时候学过计概C，觉得对计算机比较感兴趣，于是选修了计概B。

平时的月考和模考基本上都在AC3-AC4之间徘徊，期末机考AK，大概用时一小时多一点，在OJ排行榜上排到了第五名，是一个超级出乎意料的成绩。主要是由于期末题目做起来确实很顺手，前五道都是一遍过，最后一道题也是看一眼就有思路（之前做蛇入迷宫的内存优化还有每日选作里面的小游戏的时候就有过DP+BFS的做法，尤其是小游戏那道题里面DP记录最短线段数的做法），但是由于考场上比较唐诗，把BFS里面的剪枝写成了break而不是continue，导致debug了很久，后来在修改的过程中删了一个剪枝条件，导致最后提交还吃了两发罚时，但是总体来看期末题做下来确实特别顺手，基本没有遇上思路上的明显卡壳，可以说是很超常和出乎意料的发挥。

虽然笨人在经管学部，是文理兼收的，但是在高中的时候确实是学的纯文，所以在选这门课的时候，我就做好了有可能托绩点的准备，因为我自认为脑子还是不如群里面的大佬好的，尤其是每天看到群里面每天都在讨论的各种难题和算法，初期的时候我完全看不懂，等到后面才发现自己在刷的题可能群里的大佬在几个月之前就已经做过并且讨论过了，更加能感觉到差距；但是也是因此我学这门课付出了超常的努力和时间，可以说每周花在计概上的时间和其他课的时间加起来差不多了，考后数了数，我做的题目加起来大约有300+，包括每日选作全部，计算机思维算法里面除了部分T和重复题目，以及CF,leetcode,洛谷等等上面一些比较杂的题目。

如果要说有什么考高分的方法的话，我觉得最重要的就是要多做题，尤其是针对零基础或者先前接触编程少的同学。编程这种东西是越做越有的，刷题的数目不存在上限，尽管说机考的时候也存在某些神秘WA和报错，但是在刷题量足够多的情况下，机考稳定AK确实很难，但是想要得高分还是可以做到的；另外要学会善用AI，不会的算法或者知识点可以问AI原理，不过也不能过度依赖，要培养自己debug的能力，不然考试的时候遇到WA查不出来容易心态炸裂，至于应该怎么权衡只能说暂时不能给你明确答复（

另外一定要保持良好的心态，一定要对自己有信心，正视自己和大佬的差距，巨大的差距确实是客观存在的，但是群里很多大佬的能力其实是远超机考AK的要求的，并不一定要学的像群里大佬那么强；机考的时候一定也要保持心态（这点在我cheatsheet最后也写了，也是对自己的提醒，尽管最后cheatsheet似乎都没咋用），不要慌张；努力刷题总结，最后一定能取得理想的成绩的！
